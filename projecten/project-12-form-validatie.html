Project 12: Form Validatie

Advanced

Valideer een formulier met real-time feedback.

📚 Wat ga je leren?

In dit project leer je werken met form validatie, regular expressions en preventDefault. Je leert:

  • Regular expressions (regex) gebruiken voor email validatie
  • preventDefault() gebruiken om form submit te voorkomen
  • Real-time validatie met blur events
  • Error messages dynamisch tonen en verbergen
  • Visual feedback geven met CSS styling

🎯 Belangrijke Concepten

  • Regular Expressions (regex) - Patronen om tekst te matchen (/pattern/)
  • test() - Methode om te controleren of string matcht met regex
  • preventDefault() - Voorkomt standaard browser gedrag (zoals form submit)
  • blur event - Event dat afgaat wanneer element focus verliest
  • nextElementSibling - Volgende element opzelfde niveau in DOM

📝 Stap-voor-Stap Uitleg

Stap 1: Validatie functie maken

Maak functie die input valideert en feedback geeft:

function validateNaam() {
    const error = naamInput.nextElementSibling;
    if (naamInput.value.length < 3) {
        error.textContent = 'Naam moet minimaal 3 karakters zijn';
        naamInput.style.borderColor = 'red';
        return false;
    }
    error.textContent = '';
    naamInput.style.borderColor = 'green';
    return true;
}

Uitleg: Functie controleert lengte, toont error message, geeft visuele feedback en retourneert boolean.

Stap 2: Email validatie met regex

Gebruik regular expression voor email validatie:

function validateEmail() {
    const error = emailInput.nextElementSibling;
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    if (!emailRegex.test(emailInput.value)) {
        error.textContent = 'Voer een geldig email adres in';
        emailInput.style.borderColor = 'red';
        return false;
    }
    error.textContent = '';
    emailInput.style.borderColor = 'green';
    return true;
}

Uitleg: Regex controleert of email formaat geldig is: tekst@tekst.tekst zonder spaties.

Stap 3: Blur event listeners

Valideer wanneer gebruiker veld verlaat:

naamInput.addEventListener('blur', validateNaam);
emailInput.addEventListener('blur', validateEmail);

Uitleg: blur event gaat af wanneer gebruiker veld verlaat, ideaal voor validatie.

Stap 4: Form submit voorkomen

Voorkom submit als validatie faalt:

form.addEventListener('submit', function(e) {
    e.preventDefault();
    if (validateNaam() && validateEmail()) {
        alert('Formulier is geldig!');
    }
});

Uitleg: preventDefault() voorkomt standaard form submit. Alleen bij geldige input wordt actie uitgevoerd.

const form = document.getElementById('validatie-form');
const naamInput = document.getElementById('naam-input');
const emailInput = document.getElementById('email-input');

function validateNaam() {
    const error = naamInput.nextElementSibling;
    if (naamInput.value.length < 3) {
        error.textContent = 'Naam moet minimaal 3 karakters zijn';
        naamInput.style.borderColor = 'red';
        return false;
    }
    error.textContent = '';
    naamInput.style.borderColor = 'green';
    return true;
}

function validateEmail() {
    const error = emailInput.nextElementSibling;
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    if (!emailRegex.test(emailInput.value)) {
        error.textContent = 'Voer een geldig email adres in';
        emailInput.style.borderColor = 'red';
        return false;
    }
    error.textContent = '';
    emailInput.style.borderColor = 'green';
    return true;
}

naamInput.addEventListener('blur', validateNaam);
emailInput.addEventListener('blur', validateEmail);

form.addEventListener('submit', function(e) {
    e.preventDefault();
    if (validateNaam() && validateEmail()) {
        alert('Formulier is geldig!');
    }
});

✅ Instructies:

  • Maak validatie functies die boolean retourneren
  • Gebruik nextElementSibling om error element te vinden
  • Gebruik regex voor email validatie
  • Voeg blur event listeners toe voor real-time validatie
  • Gebruik preventDefault() om submit te voorkomen

💡 Tips & Tricks

  • Regex uitleg: /^[^\s@]+@[^\s@]+\.[^\s@]+$/ betekent: tekst zonder @ of spatie, dan @, dan tekst, dan ., dan tekst
  • blur vs input: blur valideert bij verlaten veld, input bij elke toetsaanslag
  • HTML5 validatie: Gebruik ook HTML5 attributen zoals required, pattern, minlength
  • Accessibility: Koppel error messages aan inputs met aria-describedby

⚠️ Veelvoorkomende Fouten

Fout 1: Vergeten preventDefault

// ❌ FOUT - Form wordt altijd gesubmit
form.addEventListener('submit', function(e) {
    if (validateNaam()) { ... } // Vergeet preventDefault!
});

// ✅ GOED - Voorkom standaard submit
form.addEventListener('submit', function(e) {
    e.preventDefault();
    if (validateNaam()) { ... }
});

Fout 2: Verkeerde regex syntax

// ❌ FOUT - String in plaats van regex
const emailRegex = '/^[^\s@]+@[^\s@]+\.[^\s@]+$/'; // String!

// ✅ GOED - Gebruik /.../ voor regex
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

Fout 3: Vergeten error message te clearen

// ❌ FOUT - Error blijft staan bij geldige input

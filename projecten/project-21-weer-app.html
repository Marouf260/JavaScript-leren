Project 21: Weer Applicatie

Master

Maak een moderne API-gestuurde weer app met een "glassmorphism" ontwerp.

📚 Wat ga je leren?

In dit project combineren we moderne UI technieken met data ophalen (simulatie). Je leert hoe je:

  • Een Glassmorphism effect maakt met CSS (backdrop-filter)
  • Functies bouwt om input van de gebruiker te verwerken
  • Simulatie API requests maakt om weersgegevens op te halen
  • De DOM dynamisch bijwerkt met de opgehaalde data

🎯 Belangrijke Concepten

  • backdrop-filter: blur() - Creëert het matglazen effect voor moderne UIs
  • Async/Await patronen - Begrijpen hoe je data in de toekomst ophaalt (hier gesimuleerd)
  • Object Destructuring - Makkelijker waarden uit objecten halen (bijv. const { temp } = weersData;)

📝 Stap-voor-Stap Uitleg

Stap 1: HTML Elementen Selecteren

Nadat we de strakke CSS hebben gedefinieerd, halen we de benodigde elementen op in JS om ze later te updaten.

const searchBtn = document.querySelector('.search-btn');
const searchBar = document.querySelector('.search-bar');

Uitleg: We gebruiken querySelector om elementen op basis van hun class (een punt .) te selecteren.

Stap 2: Zoekfunctionaliteit toevoegen

We binden een click event aan de zoekknop en een keyup (Enter) event aan het invoerveld.

searchBtn.addEventListener('click', function () {
    weatherObj.search();
});

searchBar.addEventListener('keyup', function (event) {
    if (event.key == "Enter") weatherObj.search();
});

Uitleg: Het koppelen van acties aan zowel de zoekknop als de 'Enter' toets maakt de app veel gebruiksvriendelijker.

Stap 3: Mock Data ophalen en tonen

Omdat we hier geen echte API-sleutel vereisen, maken we een "nep" database (Object) van steden. Als de stad gevonden is, updaten we de elementen.

const mockData = {
    'amsterdam': { temp: 18, desc: 'Licht bewolkt', hum: 65, wind: 12 }
};

const data = mockData[city.toLowerCase()];
if(data) {
    document.querySelector('.city').innerText = "Weer in " + city;
    document.querySelector('.temp').innerText = data.temp + "°C";
}

Uitleg: We simuleren een database en we halen de juiste data op met de bracket-notatie: object[key].

💻 Complete Code (JS Object Structuur)

const weatherObj = {
    mockData: {
        'amsterdam': { temp: 18, desc: 'Licht bewolkt', hum: 65, wind: 12 },
        'london': { temp: 15, desc: 'Regenachtig', hum: 82, wind: 20 }
    },
    fetchWeather: function (city) {
        const data = this.mockData[city.toLowerCase()];
        if (data) {
            this.displayWeather(city, data);
        } else {
            alert("Stad niet gevonden!");
        }
    },
    displayWeather: function (city, data) {
        document.querySelector('.city').innerText = "Weer in " + city;
        document.querySelector('.temp').innerText = data.temp + "°C";
    },
    search: function () {
        this.fetchWeather(document.querySelector('.search-bar').value);
    }
};

✅ Jouw Opdracht:

  • Bestudeer de CSS backdrop-filter eigenschap in de broncode om te begrijpen hoe het glas-effect werkt.
  • Probeer in de demo de steden "Amsterdam", "London", "Parijs", of "Tokyo" te zoeken.
  • Breid (lokaal) de mockData uit met jouw eigen woonplaats!

💡 Tips & Tricks

  • Waarom Object-georiënteerd? Door functies en data in weatherObj te groeperen, hou je de globale scope schoon en code overzichtelijk.
  • toLowerCase(): Zoekopdrachten moeten case-insensitive zijn (bijv. "AmsterDam"). Door de invoer om te zetten naar kleine letters via toLowerCase() voorkom je zoekfouten.
  • Echte API: Ooit klaar om een échte API te gebruiken? Vervang de mock-data door een fetch() call (zoals we doen in eerdere/latere projecten!).

⚠️ Veelvoorkomende Fouten

Fout 1: "Enter" niet herkennen

// ❌ FOUT - Verwacht een 'keyCode' die deprecated is
searchBar.addEventListener('keyup', function (event) {
    if (event.keyCode === 13) { ... }
});

// ✅ GOED - Gebruik event.key
searchBar.addEventListener('keyup', function (event) {
    if (event.key === "Enter") { ... }
});

Fout 2: Hoofdlettergevoeligheid The

// ❌ FOUT - Als de gebruiker "AmsTERdam" typt, matcht dit niet.
const data = this.mockData[city];

// ✅ GOED - Zet alles om naar kleine letters en vergelijk dan
const data = this.mockData[city.toLowerCase()];

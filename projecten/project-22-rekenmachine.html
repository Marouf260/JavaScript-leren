Project 22: Neumorphic Rekenmachine

Master

Bouw een volledig functionele rekenmachine met een prachtig, 3D 'neumorphism' ontwerp.

📚 Wat ga je leren?

We bouwen een elegante rekenmachine waar state (display waarde) correct geüpdatet moet worden.

  • Neumorphism CSS tricks met dubbele box-shadow
  • Het afhandelen van ingewikkelde events (operators, clear, evalueren)
  • Fouten afvangen wanneer een berekening niet klopt (bijv. delen door nul)

🎯 Belangrijke Concepten

  • eval() of vergelijkbaar - Functie om strings als JS logica te berekenen
  • Event Delegation - In plaats van 16 listeners te binden, binden we er 1 op de container!

📝 Stap-voor-Stap Uitleg

Stap 1: Variabelen instellen

We hebben een verwijzing naar het display nodig en een "flag" (boolean) om te weten wanneer we het display na een rekensom moeten opschonen.

const display = document.getElementById('display');
let shouldResetDisplay = false;

Uitleg: shouldResetDisplay voorkomt dat je bijvoorbeeld "12" + "5" = "17" en daarna "8" typt, en het display "178" toont in plaats van opnieuw te beginnen met "8".

Stap 2: Getallen toevoegen aan de som

Wanneer een gebruiker op een getal-knop klikt, wordt dit achter de huidge tekst geplakt.

function appendNumber(number) {
    if (display.innerText === '0' || shouldResetDisplay) {
        display.innerText = number;
        shouldResetDisplay = false;
    } else {
        display.innerText += number;
    }
}

Uitleg: Als het display "0" is of net gereset, overschrijven we het. Anders plakken (+=) we het getal erachter. Zelfs decimalen werken zo!

Stap 3: Veilig "Evalueren" (Berekenen)

Wanneer de gebruiker op `=` drukt, moeten we de tekstuele som berekenen (bijvoorbeeld "14*2+5"). Dat is magie in JS met de new Function of eval() methodes.

try {
    const result = new Function('return ' + display.innerText)();
    display.innerText = Math.round(result * 1000000) / 1000000;
} catch (error) {
    display.innerText = 'Error';
}

Uitleg: We vangen vreemde invoer op in de catch blok zodat de browser niet crasht. Tevens ronden we JS-floating-point-bugs (0.1 + 0.2 = 0.3000004) vakkundig af.

💻 Code Snippet (Operatoren Voorkomen)

function appendOperator(op) {
    // Wat als de gebruiker twee keer op "+" klikt? (bijv. "5++")
    const lastChar = display.innerText.slice(-1);
    
    // Is het laatste karakter al een operator? Overschrijf hem!
    if (['+', '-', '*', '/', '%'].includes(lastChar)) {
        display.innerText = display.innerText.slice(0, -1) + op;
    } else {
        display.innerText += op;
    }
}

✅ Jouw Opdracht:

  • Klik op de knoppen in de demo en voer een complexe berekening uit.
  • Voel hoe de knoppen indrukken dankzij het "Inset Box Shadow" (Neumorphism) CSS effect.
  • Bekijk hoe de pijl ← direct het laatste karakter verwijdert dankzij de `slice(0, -1)` methode.

💡 Tips & Tricks

  • Waarom new Function() i.p.v. eval()? In een professionele web-omgeving is eval() dodelijk vanwege veiligheidsrisico's (XSS/Injections). De new Function methode creëert een anonieme functie in een sandbox die geen toegang heeft tot lokale scope en geldt als veiliger alternatief, maar blijf opletten bij directe verwerking van User-Input!
  • Grote Cijfers (Text-overflow): Kijk in de CSS naar de `.display` class met text-overflow: ellipsis;, zodat een resultaat van "10423403248324.9" niet je neomorphism-buidel "breekt".

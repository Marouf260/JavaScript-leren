Project 25: Uitgaven Tracker

Master

Bouw een uitgebreide tracker die budget en uitgaven lokaal opslaat.

📝 Stap-voor-Stap Uitleg

Stap 1: Local Storage Begrijpen

Browsers hebben een mini-database genaamd localStorage. Data overleeft hiermee een overlaad (F5) van de pagina. Omdat localStorage alleen tekst opslaat, gebruiken we JSON.stringify() en JSON.parse() om arrays te vertalen.

const localStorageKey = 'tracker_project25_data';
const savedData = JSON.parse(localStorage.getItem(localStorageKey));

let transactions = savedData !== null ? savedData : [];

Uitleg: We kijken of we al data hebben opgeslagen. Zo ja, dan laden we dat (als echte array d.m.v. parse), anders starten we met een lege array ([]).

Stap 2: Wiskunde met Arrays `.reduce()`

In theorie kunnen we met een for loop door alle transacties gaan om het totaal te berekenen. Echter, .reduce() leent zich hier perfect voor: het "reduceert" een array naar één waarde.

const amounts = transactions.map(t => t.amount);
const total = amounts.reduce((acc, item) => (acc += item), 0);

Uitleg: acc (accumulator) is de "spaarvarken" variabele die start bij 0 (tweede argument) en bij elke ronde het item (bedrag) erbij optelt.

Stap 3: Het Formulier Afvangen

Wanneer een gebruiker het formulier verstuurt, vangen we de gebeurtenis op (zodat de pagina niet herlaadt!) en voegen we de data toe aan de state.

function addTransaction(e) {
    e.preventDefault(); // Voorkom form Reload!
    
    if (text.value.trim() === '') return; // Valideer!
    const transaction = { id: Math.random(), text: text.value, amount: +amount.value };
    
    transactions.push(transaction);
    updateLocalStorage();
}

Uitleg: We maken een nieuw object via destructuring, we slaan die op in de array, en we updaten de localStorage.

💻 Code Snippet (Verwijderen o.b.v. ID)

function removeTransaction(id) {
    // We filteren simpelweg de array zodat alles behalve het ID-target overblijft
    transactions = transactions.filter(transaction => transaction.id !== id);
    
    updateLocalStorage();
    init(); // Teken de DOM (het scherm) weer up-to-date!
}

✅ Jouw Opdracht:

  • Vul een item ("Boodschappen") en een negatief bedrag in ("-30").
  • Ververs (F5) de pagina. Jouw items zijn er nog steeds! (Dankzij de browser opslag localStorage).
  • Hover (met je muis) over de lijstitems heen en druk op de rode "X" om ze weer te verwijderen.

💡 Tips & Tricks

  • De plus prefix +amount.value: HTML textinput fields retourneren altijd strings (zelfs als je type="number" gebruikt). De optionele plus + voor de variabele in javascript dwingt de data om zich te gedragen als een Number in plaats van tekst.
  • Waarom een ID in de structuur? Zonder uniek ID kun je in removeTransaction niet heel betrouwbaar de object index opzoeken en opschonen. Stel de gebruiker heeft 2x een transactie staan van "Pizza" met "15,50", welke moet dan weg?

⚠️ Veelvoorkomende Fouten

Fout 1: JSON.parse() vergeten

// ❌ FOUT - LocalStorage returnt de data ALTIJD als tekst (een string), pushen hierop zal crashen!
let data = localStorage.getItem('mijn_key');
data.push({ ... }); 

// ✅ GOED - Vertel Javascript dat de "string" data uit LocalStorage in realiteit een Array en Object is!
let data = JSON.parse(localStorage.getItem('mijn_key'));

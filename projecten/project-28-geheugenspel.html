Test je Javascript en CSS vaardigheden door een interactief geheugenspel met 3D animaties te bouwen.

📝 Stap-voor-Stap Uitleg

Stap 1: De Kaarten Spawner (Array Duplicatie)

Een geheugenspel heeft paren nodig. In plaats van in HTML of JS elke kaart driedubbel in te typen, maken we 1 basislijst aan in een JavaScript array. Vervolgens gebruiken we de Spread Operator (...) om deze array in één klap te dupliceren en daarna te shufflen.

const cardsArray = [{ name: 'vuur', icon: '🔥' }, { name: 'water', icon: '💧' }];

// Resultaat: [vuur, water, vuur, water]
let gameGrid = [...cardsArray, ...cardsArray];

Uitleg: De syntaxis [...array1, ...array1] pakt alle elementen uit en stopt ze achter elkaar in een compleet nieuwe doos / geheugenruimte.

Stap 2: The Flip (CSS Animatie & State)

Als we op een kaart klikken, gebeurt er in JavaScript eigenlijk vrij weinig magisch voor de 3D weergave: we voegen slechts de "flip" class toe. De CSS eigenschap transform: rotateY(180deg) doet uitsluitend de styling rest. 

function flipCard() {
    // Block the kliks als animaties nog bezig zijn
    if (lockBoard) return; 
    
    this.classList.add('flip');
}

Uitleg: De lockBoard variabele is een heel belangrijke state-boolean. Het voorkomt dat de gebruiker gefrustreerd een 3e kaart omdraait, terwijl de 2 foute kaarten nog visueel met een animatie bezig zijn terug te keren.

Stap 3: setTimeout() (Tijdvertragend Terugdraaien)

Stel de speler heeft 2 kaarten aangeklikt die niet overeenkomen. Dan wil je de kaarten niet IN ÉÉN milliseconde meteen weer omdraaien! Anders ziet de speler het tweede plaatje immers niet eens en is het spel te pijnlijk moeilijk.

function unflipCards() {
    lockBoard = true; // 🔒 Zet the bord the the tijdelijk op slot

    setTimeout(() => {
        firstCard.classList.remove('flip');
        secondCard.classList.remove('flip');
        
        resetBoard(); // 🔓 Geef clicks the the weer vij
    }, 1000); // Wacht 1000 milliseconden (1s) the the
}

Uitleg: setTimeout() is asynchroon; de code-blok erin wordt pas 1 seconde opgeschoven in de toekomst gelanceerd.

✅ Jouw Opdracht:

  • Speel het spel hieronder! Het bord genereert de kaarten elke keer in een willekeurige volgorde ("shuffle").
  • Let op hoe vloeiend de "Flip" 3D animatie gaat.
  • Pauzeer even wanneer je er twee fout hebt; je kunt in de test-omgeving tijdelijk géén nieuwe kaarten doorklikken totdat unflipCards klaar is.

💡 Tips & Tricks

  • Waarom dataset? Als we <div data-icon="water"> schrijven in de HTML, kunnen we dit in JavaScript heel gemakkelijk uitlezen met element.dataset.icon. Het is verreweg de schoonste manier om onzichtbare ID's of data in de browser mee te geven aan een DOM element.
  • The This Keyword: De this variabele in de flipCard functie is een heel dynamisch (en berucht) woord in JavaScript! In de context van addEventListener('click', flipCard), verwijst this altijd automatisch naar het element waarop exact geklikt is.

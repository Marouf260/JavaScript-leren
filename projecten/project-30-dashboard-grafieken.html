📝 Stap-voor-Stap Uitleg

Stap 1: De "Procentuele Wiskunde" (Max Waarde Berekenen)

Als je een staafdiagram wilt maken met flexibele hoogtes, moet je eerst weten wat je referentiepunt is (de 100% kapstok). We doen dit door met de power API Math.max() de allerhoogste waarde in onze lijst met data te vinden.

// 1. Array extractie m.b.v de Spread Operator
// 2. [30, 45, 95...] wordt hier 'uitgepakt' en vergeleken!
const maxVal = Math.max(...chartData.map(d => d.value));
// Uitkomst in dit script: maxVal = 95

Uitleg: Als we weten dat het hoogste cijfer (bijvoorbeeld 95k in juli) ons plafond is, kunnen alle andere maanden dynamisch via een percentageberekening (zoals (huidig / 95) * 100) visueel geproportioneerd worden.

Stap 2: Elementen Creëren en Tekenen

Zodra we de data hebben omgerekend, bouwen we de grafiek visueel op in onze HTML chartContainer. Voor elk data item (elke maand) in de loop, bouwen we een container div inclusief tooltip, bar div en label div. 

chartData.forEach(item => {
    // Maak de container (vergelijkbaar met document.createElement('div'))
    const container = document.createElement('div');
    container.className = 'bar-container';

    // Injecteer de statische HTML elementen MET 0% beginhoogte
    container.innerHTML = `
        <div class="bar-tooltip">${item.value}K</div>
        <div class="bar" style="height: 0%"></div>  
        <div class="bar-label">${item.label}</div>
    `;
    chartContainer.appendChild(container);
});

Uitleg: We starten bewust met een element hoogte van 0%. De animatie groeit namelijk sneller visueel in setTimeout() wanneer de CSS engine begrijpt dat het een overgang van 0 tot X% opzet.

Stap 3: Het CSS Dark/Light Thema Wisselen

Tegenwoordig heeft elke professionele app een Donkere Modus. Met een boolean variable houden we in JS gemakkelijk de status van de app bij en 'togglen' we de interface classes die op de main parent body/dashboard zijn gedeclareerd.

themeBtn.addEventListener('click', () => {
    isDark = !isDark; // Flip (Inverteer de booolian True/False statement)
    
    if (isDark) {
        dashArea.classList.add('dark-theme');
        themeBtn.innerText = '☀️';
    } else {
        dashArea.classList.remove('dark-theme');
        themeBtn.innerText = '🌙';
    }
});

Uitleg: In de CSS zie je regels zoals .project-area.dark-theme .stat-card. Als we globaal via knop events de parent class aansturen, passen alle child cards, navbars en kleuren zich direct proactief door de cascading style layout perfect aan.

✅ Jouw Opdracht:

  • Klik rechtsboven op het thema-icoontje (Zon/Maan) voor "Dark Mode". Alle dashboard componenten verkleuren automatisch d.m.v global parent states.
  • Houd je cursor (hover) over de speelse blauwe staven van de grafiek onderaan om de individuele datapunten (de tooltips) visueel in te zien.

💡 Tips & Tricks

  • Waarom geen Chart.js Library? Voor snelle, lichte prototypes is een element query via simpele CSS vaak honderden kilobytes "lichter" om te laden voor een browser over netwerken, en een perfecte manier om creatieve inzichtelijke wiskunde logica te bouwen in JavaScript!
  • position: relative & position: absolute: Om de "tooltips" exact BOVEN de staaf te positioneren, geven we de .bar-container parent de statische eigenschap relative en de tooltip class de absolute. Zo zweeft de tooltip relatief! t.o.v zijn eigen staaf.

// ❌ FOUT - Als je de CSS .bar-chart class parent (de grijze base border layout) niet voorziet van een vaste height (bijv. 150px), dan weten je .bar kind elementen met hun 100% height parameters helemaal NIET HOOG ze moeten worden! Ze vallen dan plat naar 0px.

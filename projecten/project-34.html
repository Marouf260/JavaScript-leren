Project 34: Multi-Step Formulier

Advanced

Maak een onboarding-style formulier dat input over meerdere "slides" of stappen verdeelt.

📝 Stap-voor-Stap Uitleg

Stap 1: HTML Structuur Opzetten

We maken één grote "form-container" en daarbinnen meerdere onafhankelijke <div class="form-step"> elementen. Standaard verbergen we al deze elementen in de CSS (display: none), BEHALVE degene die de .active CSS class heeft!

<!-- de CSS verbergt automatische alle form-steps die GEEN 'active' class HEBBEN! -->
<div class="form-step active"> Stap 1 forms </div>
<div class="form-step"> Stap 2 forms </div>

Uitleg: Dit concept heet "conditional rendering". De JS bepaalt later welke div de active class krijgt! De rest blijft onzichtbaar.

Stap 2: JS "Current Step" State bijhouden

Om te weten welke stap in de DOM we moeten tonen, hebben we een variabele nodig om de status van ons formulier bij te houden!

// De state! 0 = Stap 1, 1 = Stap 2, 2 = Stap 3!
let currentStep = 0;

nextBtn.addEventListener('click', () => {
    // Als we nog niet bij de laatste stap zijn:
    currentStep++; // Verhoog de step!
    updateFormUI(); // Voer de UI aanpassingen nu uit!
});

Uitleg: JS is de 'beheerder' van de formuliere-applicatie. De code baseert altijd zijn weergaves en beslissingen op de waarde van deze ene "state" variabele.

Stap 3: De UI Vernieuwen

Als currentStep is geüpdatet, moeten we de DOM elementen met de CSS classes van de stappen up-to-date maken!

function updateFormUI() {
    // 1. Maak alles onzichtbaar
    formSteps.forEach(step => step.classList.remove('active'));
    
    // 2. Voeg de active class toe aan de nieuwe huidige step!
    formSteps[currentStep].classList.add('active');
}

Uitleg: Dit is de kracht van de the Array `forEach` methode. We doen eerst een "reset" door de CSS active classes meedogenloos bij allemaal te verwijderen. Daarna geven we alleen de juiste huidige active class netjes terug op basis van the `currentStep` index!

💻 Code Snippet (Button the States)

// Om bugs te voorkomen kunnen we de 'terug' knop uitschakelen op de eerste stap! 
if (currentStep === 0) {
    prevBtn.disabled = true; // Vorige knop is onklikbaar!
} else {
    prevBtn.disabled = false;
}

✅ Jouw Opdracht:

  • Vul het formulier in en klik op de "Volgende" knop.
  • Zie hoe de bolletjes bovenin bijgewerkt worden met de juiste status!
  • Probeer ook terug te gaan met de "Vorige" knop!

💡 Tips & Tricks

  • formSteps.length: In de controle om te bepalen of we bij de laatste stap zitten, maken we slim gebruik van `formSteps.length - 1` in plaats van hardcoded een the gets `2` erin the the thezetten. Dit maakt je code dynamischer, je kan nu makkelijk een 4e HTML sectie the the then toevoegen!

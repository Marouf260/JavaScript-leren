Project 36: Wachtwoord Sterkte Meter

Advanced

Maak een robuuste client-side validatietool die de veiligheid van een wachtwoord toetst.

📝 Stap-voor-Stap Uitleg

Stap 1: Reguliere Expressies Leren Kennen

Reguliere Expressies, vaak afgekort tot RegEx, vormen een mini-taaltje waarmee je patronen in tekst kunt zoeken. Het ziet er vaak uit als ingewikkelde computercode, maar is logisch opgebouwd.

// Zoek naar in ieder geval 1 hoofdletter:
const heeftHoofdletter = /[A-Z]/.test("mijnWachtwoord"); // → true

// Zoek naar een cijfer (0 t/m 9):
const heeftCijfer = /[0-9]/.test("geheim"); // → false

Uitleg: We gebruiken /patroon/.test(string) in JavaScript. Deze functie retouneert altijd vliegensvlug een simpele true of false. Dat is ideaal voor wachtwoord validatie!

Stap 2: Het 'Input' Event Luisteren

We moeten elke keer dat de gebruiker iets intypt, ons validatie-script opnieuw runnen. We gebruiken hiervoor niet keyup of keydown, maar het input event!

wachtwoordVeld.addEventListener('input', (event) => {
    // Lees de actuele ingetypte waarde uit het veld
    const text = event.target.value;
    
    // Voer de RegEx checks op deze tekst uit!
    checkWachtwoord(text);
});

Uitleg: Het input event is perfect. Waarom? Omdat dit event ook afgaat als de gebruiker met de rechtermuisknop een wachtwoord 'plakt', iets wat de toetsenbord-events (keyup/down) volledig missen!

Stap 3: Het CSS Score Systeem

Elke keer als het wachtwoord aan een eis voldoet (bijv. lente > 8 of een cijfer), voegen we een punt toe aan de score (maximaal 4 punten). Die score vertalen we naar een CSS percentage.

function updateMeter(score) {
    // Score gedeeld door max score * 100 = Percentage
    let breedte = (score / 4) * 100;
    meterBar.style.width = `\${breedte}%`;
    
    if (score === 4) {
        meterBar.style.backgroundColor = '#10b981'; // Groen = veilig!
    }
}

Uitleg: Hoe hoger de score, hoe langer het CSS gekleurde balkje wordt. We veranderen bij de topscore ook het kleurtje naar geruststellend groen.

✅ Jouw Opdracht:

  • Typ een simpel wachtwoord in de demo (zoals 'wachtwoord'). De score-balk reageert en blijft rood/zwak.
  • Voeg nu hoofdletters, cijfers en vreemde karakters (zoals #, $, @ of !) toe.
  • Merk op hoe de dynamische bulletpoints live afvinken en de balk naar 100% Groen springt!

💡 Tips & Tricks

  • Gevoelige Data Feedback: Vertel een gebruiker altijd waarom een wachtwoord afgekeurd wordt. Dat is de reden achter de live afvinkende bulletpoints. Dat is véél betere UX (User Experience) dan een vage "Wachtwoord te zwak" popup achteraf!

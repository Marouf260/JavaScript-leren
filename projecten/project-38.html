Project 38: Toast Notificaties

Advanced

Vervang lelijke `alert()` boxen met een mooi geanimeerd, stapelbaar toast systeem.

📝 Stap-voor-Stap Uitleg

Stap 1: De Toast Container

Toasts (of snackbars) zijn tijdelijke berichtjes die in beeld schuiven. Omdat het er meerdere tegelijk kunnen zijn, plaatsen we ze in een vaste, onzichtbare poortwachter: een #toast-container die rechtsonder in het scherm vastgemaakt ("fixed") is.

/* De container staat vast op het scherm */
#toast-container {
    position: fixed;
    bottom: 20px;
    right: 20px;
}

Uitleg: Vanaf deze plek kunnen we met Javascript dynamisch zoveel berichtjes injecteren als we willen, en ze zullen netjes boven elkaar 'stapelen'.

Stap 2: Elementen Genereren in JS (createElement)

In plaats van een HTML div onzichtbaar te maken, genereren we een compleet nieuwe HTML div on the fly met document.createElement('div') en injecteren we de HTML string daarbinnen.

function showToast(bericht) {
    // Maak een lege <div>
    const toast = document.createElement('div');
    toast.className = 'toast success'; // Geef het classes voor de CSS
    
    // Vul de div met inhoud
    toast.innerHTML = `<span>\${bericht}</span>`;
    
    // Stop hem in de container, waardoor hij op het scherm verschijnt!
    container.appendChild(toast);
}

Uitleg: Met createElement en appendChild bouw je als een LEGO-master elementen uit het niets op en klik je ze live vast in je document. Ze schuiven dan prachtig in beeld dankzij onze CSS animatie.

Stap 3: Garbage Collection (Verdwijnen)

Wat we bouwen vergaat niet zomaar. We moeten de toasts ook weer netjes van de DOM verwijderen, anders zouden er onzichtbare (transparante) divs in het geheugen ophopen! We programmeren een automatische sloop ("auto-destruct") actie na 3 seconden met setTimeout.

// Wacht 3000ms en start de verdwijn animatie (.hide)
setTimeout(() => {
    toast.classList.add('hide');
    
    // Wacht tot de slideOut animatie klaar is (300ms) en VERWIJDER hem!
    setTimeout(() => { toast.remove(); }, 300);
}, 3000);

Uitleg: Gooi altijd elementen weg met .remove() als je ze niet meer nodig hebt, zo voorkom je memory leaks (geheugenlekken)! We moeten heel even wachten (die 300ms) om te zorgen dat gemaakte CSS afgespeeld is, daarna is het tabee.

💻 Code Snippet (Handmatig Sluiten)

// Je kan ook een kruisje toevoegen dat het element handmatig wist.
// De logica zoekt het kruisje specifiek *binnen* in deze nieuwe toast!
const closeBtn = toast.querySelector('.close-toast');

closeBtn.addEventListener('click', () => {
    // Handmatig het element verwijderen. Zelfde principe!
    removeToast(toast);
});

💡 Tips & Tricks

  • Re-usable functies: Je hebt nu 1 showToast(type, cssClass) functie gebouwd. Je kan deze logica zo hergebruiken bij elk formulier dat je bouwt in de toekomst!
  • UX Feedback: Vertel een gebruiker altijd wat er gebeurt na een abstracte actie als het opslaan van code. Er is niks ongemakkelijker dan op een opslaan-button klikken en... niks zien, onzeker of het nou gefixed is of niet.

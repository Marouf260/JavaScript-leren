Project 39: Google-style Zoek Autocomplete

Advanced

Maak een dropdown lijst die zoeksuggesties geeft op het moment dat je typt (Net als Google).

📝 Stap-voor-Stap Uitleg

Stap 1: Luisteren naar Input

We luisteren naar het input event op het zoekveld. Elke keer als de gebruiker een letter typt of weghaalt, wordt dit event afgevuurd. We slaan de getypte waarde op en gebruiken `trim()` om onnodige spaties aan het begin en eind te verwijderen.

searchInput.addEventListener('input', (event) => {
    // Haal witruimtes weg aan begin/eind
    const getyptVerhaal = event.target.value.trim();
    
    if (getyptVerhaal) {
        // Zoek logica hier...
    } else {
        // Verberg popup als veld leeg is
        suggBox.classList.remove('active');
    }
});

Stap 2: De Lijst Filteren

We hebben een Array (lijst) met alle landen. We gebruiken de krachtige .filter() methode om alleen de landen te behouden die de letters bevatten die de gebruiker heeft getypt.

// We maken zowel de input als de landnamen kleine letters (.toLowerCase)
// Zodat "Ned" matcht met "nederland"
gefilterdeLijst = alleLanden.filter(land => {
    return land.toLowerCase().includes(getyptVerhaal.toLowerCase());
});

Uitleg: filter() loopt door elk item in de 'alleLanden' array. Als de functie binnenin true teruggeeft, blijft het item behouden; anders wordt het weggelaten uit de nieuwe 'gefilterdeLijst'.

Stap 3: Tekst Dikgedrukt Maken (Highlighting)

Om te laten zien wáárom een land als resultaat terugkomt, maken we het gedeelte dat overeenkomt dikgedrukt. Hiervoor gebruiken we een Reguliere Expressie (RegEx) en de .replace() methode.

// Maak een dynamische Regex van de input, g=globaal, i=case-insensitive
const regex = new RegExp(`(\${getyptVerhaal})`, "gi");

// $1 refereert naar de match in de haakjes () van de regex
const highlightedMatch = origineelWoord.replace(regex, `<span class="highlight">$1</span>`);

Stap 4: Weergeven en Event Delegation

Na het filteren en highlighten voegen we de HTML toe aan de suggestie box en maken we deze zichtbaar. In plaats van op elke <li> een click-listener te zetten, zetten we één listener op de parent <ul> (Event Delegation).

suggBox.addEventListener('click', (event) => {
    // .closest('li') zoekt naar parent 'li' of het element zelf als het een 'li' is.
    // Dit is nodig omdat je soms op de <span class="highlight"> klikt!
    const geklikteLi = event.target.closest('li');
    
    if (geklikteLi) {
        // .innerText negeert HTML tags en geeft alleen de pure tekst
        searchInput.value = geklikteLi.innerText;
        suggBox.classList.remove('active');
    }
});

✅ Jouw Opdracht:

  • Klik op het zoekveld en typ "ne" of "z".
  • Bekijk hoe de gefilterde suggesties verschijnen, met het kloppende deel van de tekst oplichtend.
  • Verwijder de tekst met backspace, en zie dat de pop-up en "active" styling (de rechte hoeken onderaan) verdwijnen.
  • Klik op een suggestie uit de lijst, en zie hoe deze naar het zoekveld wordt getransporteerd!

💡 Tips & Tricks

  • Debouncing: Als je data uit een echte database of API zou halen, is het verstandig om Debouncing te gebruiken. Dat betekent dat je een fractie van een seconde wacht na de laatste getypte toets voordat je zoekt. Anders bombardeer je de server met onnodige verzoeken bij elke letter die ingetypt wordt!

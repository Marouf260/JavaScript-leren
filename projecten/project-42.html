Project 42: RegEx Highlighter

Data Logic

Bouw een live editor voor Reguliere Expressies (RegExp) en zie direct wat jouw code matcht in de tekst!

📝 Stap-voor-Stap Uitleg

Stap 1: Dynamic Highlighting (De Illusie)

Een standaard <textarea> ondersteunt geen HTML elementen zoals <mark> of dikgedrukte kleuren er in. Hoe maken pseudo-code editors dit dan toch mogelijk? Met een optische illusie!

<!-- OP ACHTERGROND: Een transparante textarea, waar je écht in typt -->
<textarea class="test-string" spellcheck="false">Jouw tekst</textarea>

<!-- OP VOORGROND: Een DIV direct over de textarea gelegd via CSS Absolute Position -->
<!-- pointer-events: none; zorgt dat je clicks er 'doorheen' vallen.
<div class="highlights"><mark>Jouw</mark> tekst</div>

Stap 2: De RegEx Machine

Net als de letterlijke slashes /patroon/g die je typt, genereert onze Javascript live een object genaamd new RegExp().

// Lees de inputvelden uit
const patroon = regexInput.value; // bijv: "\b[A-Z]"
const flag = flagsInput.value;   // bijv: "g"

// Draai de Javascript machine aan
const regex = new RegExp(patroon, flag);

// Vervang alles in de tekst dat de RegExp raakt door een <mark> element!
// Het speciale "$&" karakter plakt het originele woord er netjes in terug.
const highlightedText = text.replace(regex, '<mark>$&</mark>');

Stap 3: Foutafhandeling (Try-Catch)

Omdat de gebruiker live RegEx typt in de inputvelden, crasht de applicatie genadeloos als the RegEx ineens ongeldig gemaakt wordt (bijvoorbeeld bij het openen van een [ maar nog niet sluiten). JavaScript probeert in paniek die incomplete regex the runnen, maar weet zich er geen raad mee.

try {
    // PROBEER dit uit te voeren...
    const regex = new RegExp(patroon, flag);
    
    // Geen error? Dan printen wij de markup in de frontend div!
    highlights.innerHTML = text.replace(regex, '<mark>$&</mark>');
    
} catch (err) {
    // Als het PROBEER gedeelte weigert te compilen, val je genadeloos hierin!
    // In plaats van de hele site te doen crashen on-screen... geven wij een sierlijke melding.
    errMsg.innerText = "Syntax Error: Je RegExp is ongeldig gebouwd.";
}

✅ Jouw Opdracht:

  • De default test zoekt naar alle woorden gestart met een letter (w+).
  • Verander de regex naar [0-9]+ om alleen getallen in de zin te omcirkelen.
  • Voer een ongeldige RegEx in, verwijder bijvoorbeeld het sluitende haakje, en verander [0-9]+ naar [0-9+ om de rode Foutmelding (Try/Catch) the testen! De website is onbeschadigd vanwege de error handling.

💡 Tips & Tricks

  • RegEx Leren: RegEx blijft voor de meesten onleesbare hiërogliefen. Maar gelukkig hoef je ze niet uit the hoofd the kennen. Maak wel alvast kennis met \d (digit/cijfer), \w (word/letter/cijfer), en the anchor ^ (begin van de string).

⚠️ Veelvoorkomende Fouten

Fout 1: Linebreaks (Enters) en HTML

// ❌ LET OP - Een textarea begrijpt dat "Enter" een newline '\n' is. Maar de gerenderde DIV begrijpt dat totaal niet en klapt alles op 1 regel. Ouderwetse truc is nodig! Vervang '\n' the RegEx expliciet door het HTML taggetje '<br>'!
const fixedLineBreaks = highlightedText.replace(/\n/g, '<br>');

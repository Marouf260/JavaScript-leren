📝 Stap-voor-Stap Uitleg

Stap 1: IntersectionObserver aanmaken

De Intersection Observer API laat je weten wanneer een element zichtbaar wordt op het scherm, zonder dure scroll-event listeners!

const observer = new IntersectionObserver(
    (entries) => {
        entries.forEach(entry => {
            if (entry.isIntersecting) {
                entry.target.classList.add('zichtbaar');
            }
        });
    },
    { threshold: 0.15 } // 15% moet zichtbaar zijn
);

Stap 2: Elementen Observeren

Na het aanmaken van de observer kun je hem koppelen aan DOM-elementen. Elke keer dat zo'n element de viewport in- of uitscrollt, wordt de callback aangeroepen.

document.querySelectorAll('.kaart').forEach(el => {
    observer.observe(el);
});

✅ Jouw Opdracht:

  • Scroll naar beneden om de animaties te triggeren!
  • Scroll daarna terug omhoog - de animaties resetten zich.
  • Bekijk de JavaScript code om te zien hoe de Observer werkt.

💡 Tips & Tricks

  • threshold: Een waarde tussen 0 en 1. 0 = zodra één pixel zichtbaar is. 1 = pas als het element volledig zichtbaar is.
  • Performance: Dit is VEEL efficiëntter dan een scroll event listener. De browser beheert intern wanneer er checks nodig zijn.

🚀 Uitbreidingsmogelijkheden

Klaar voor MEER!? Probeer dit toe te voegen:

  • Lazy Loading Images: Gebruik de Intersection Observer om afbeeldingen pas te laden als ze in beeld komen. Dit versnelt de paginalading enorm!
